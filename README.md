## ⭐ What's New in v1.0.0? (Major Update)
This release introduces a major architectural overhaul focusing on robustness, testability, and maintenance:

*   **🛡️ Robustness:** Implemented strict **Input Validation** on all public API methods to prevent invalid network requests from being generated by bad user input.
*   **🏗️ Testability:** Introduced the `APIConnector` interface via Dependency Injection. This decouples network logic from business logic, allowing for comprehensive unit and integration testing using mock servers.
*   **🧹 Maintainability:** Refactored Protobuf data mapping to centralize translation logic on response structs, dramatically reducing boilerplate code across the package.

## 🛠️ Architecture Deep Dive (For Developers)
The library's architecture has been fundamentally redesigned to adhere to modern Go patterns:

**High-Level Flow:** `Your Application` $\rightarrow$ `client/Client` $\rightarrow$ `APIConnector` $\rightarrow$ `internal/client/Conn` $\rightarrow$ `Futu OpenD Gateway`.

*   **Old Way:** `Client` held a concrete `*Conn`.
*   **New Way:** `Client` holds an abstract `APIConnector` interface. This allows for seamless swapping of the real TCP connection with a mock implementation during testing, ensuring your business logic is fully verified in isolation.
